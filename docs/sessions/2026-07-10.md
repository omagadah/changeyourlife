## Contexte
L'owner ne pouvait **plus se connecter au site** (bannière « Connexion Google réussie -
redirection… » sans redirection, cassé sur Chrome, OK sur Opera). Après le fix, carte
blanche pour enchaîner une liste de features sans re-solliciter de confirmation.

## Travaux effectués

### 1 · Fix login Google/GitHub sur Chrome (commit `3d8313a`)
- **Cause racine 1** : la redirection reposait uniquement sur `onAuthStateChanged`, qui
  ne re-fire PAS si une session est déjà persistée → popup OK mais bloqué sur la notif.
  Fix : `redirectAfterAuth(user)` appelée directement après `signInWithPopup`/redirect/
  email dans [inscription.js](../../public/js/inscription.js).
- **Cause racine 2 (le vrai blocage Chrome)** : headers serveur. Ajout dans
  [vercel.json](../../vercel.json) du header `Cross-Origin-Opener-Policy:
  same-origin-allow-popups` (débloque le popup Firebase, cf. warnings `popup.ts`) +
  `https://www.google.com` dans CSP `connect-src`/`frame-src` (`cleardot.gif` était bloqué).
- Opera était juste plus permissif → d'où le « ça marche sur Opera ». Ce n'était **pas**
  un problème de cache côté owner. Voir mémoire `login-popup-coop-csp`.

### 2 · Dark/Light toggle global dans le bandeau (commit `49ddbde`)
- `initThemeToggle()` dans [userMenu.js](../../public/js/userMenu.js) : bouton soleil/lune
  fixe à gauche du logo, sur toutes les pages authentifiées. Réutilise le système existant
  (classe `light-mode` + `localStorage 'theme'`). Event `cyf:theme-changed` pour la sync.
- ROADMAP MAJ : **audio-visualizer SYL** ajouté (module IA vocal futur, « essentiel mais
  pas central »), + fiche profil premium / dark mode / giveaway.

### 3 · Module Giveaway violet + timer iOS (commit `e3a9326`)
- [giveaway.js](../../public/js/giveaway.js) : carte **violette** (contraste avec l'app
  bleutée) injectée en haut de `/app` (`order:-7`). Timer iOS-style (J/H/M/S en tuiles,
  chiffres tabulaires) jusqu'au prochain tirage hebdo (dimanche 20:00 local).
- **Cooldown** : bouton « Participer » → participation enregistrée pour le cycle
  (localStorage par UID), état verrouillé « Tu participes » jusqu'au tirage suivant.
- 100% client-side, structure prête à brancher Firestore/CF quand le tirage devient réel.

### 4 · Fiche profil premium (commit `39c1cb5`)
- Hero card = carte d'identité avec **liseré métal argenté rotatif** (`@property
  --cyl-angle` + `conic-gradient` border-box, fallback statique), voile de brillance,
  anneau métallique autour de l'avatar. Fusion esprit *metallic-silver* + *profile-card*.
- **Upload avatar animé** [avatar-upload.js](../../public/js/avatar-upload.js) : drag &
  drop, barre de progression, vignette de preview, état « done » ✓. Le drop réinjecte le
  fichier dans `#avatar-file-input` → `profile.js` gère le downscale/sauvegarde (zéro casse).

### 5 · Fix connexion Google Agenda (commit `79f7946`)
- [agenda.js](../../public/js/agenda.js) : `connect()` ne relance plus un 2e popup quand
  l'utilisateur ferme/bloque (Chrome bloquerait) ; repli `signInWithPopup` seulement pour
  les cas légitimes. `connectErrorMessage()` = messages précis par code au lieu du
  générique « annulée/refusée » + `console.error` du code exact. Bénéficie aussi du header
  COOP. **Reste à valider** en live (peut dépendre du mode Test OAuth Google Cloud).

### 6 · Réorganisation hiérarchie d'arrivée `/app` (commit `6b6566f`)
- Ordre : giveaway (promo) → accueil → **l'ARBRE** (cœur du produit, cf. VISION, remonte
  tout en haut) → ta journée (agenda + aujourd'hui) → tes chiffres (stats + niveaux) →
  outils. L'arbre n'est plus enfoui sous les stats. Fait sans la vision précise de l'owner
  (amélioration sûre) — **à affiner** avec sa vision exacte.
